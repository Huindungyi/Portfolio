VI. Incident Response & Troubleshooting

Case 1. 대용량 데이터 누적으로 인한 MES 접속 불가

<문제>
MES 전면 무기한 로딩 발생으로 현장 업무 중단

<원인 판단>
최근 데이터 급증 시점 이후 성능 저하가 누적된 점을 근거로
누적 데이터로 인한 DB 성능 한계 가능성을 우선 가설로 설정

<조치>
서비스 일시 중단 후 공지
오래된 데이터 백업 및 단계적 삭제 수행
시스템 정상 작동 여부 우선 확보
이후 주기적 심야 데이터 정리 정책 수립
근본 원인 분석은 서비스 안정화 이후 단계적으로 진행

<결과>
MES 정상 접속 복구
업무 재개 가능 수준으로 성능 안정화

<교훈>
운영 환경에서는 완벽한 원인 규명보다
신속한 안정화와 데이터 증가를 고려한 사전 관리가 우선됨

VII. DBA Mindset & Career Direction

저는 장애나 성능 저하를 마주했을 때
증상이 아닌 데이터의 상태, 흐름, 기준부터 점검하는 방식으로 문제를 해결해 왔습니다.

MES·APS 운영 환경에서 반복적으로 확인한 사실은
문제의 대부분이 애플리케이션이 아닌 데이터 구조와 정합성 관리 부재에서 비롯된다는 점이었습니다.

이 경험을 통해 저는 DBA의 역할을
단순한 DB 운영자가 아닌, 시스템과 현장을 연결하며 서비스의 신뢰성을 책임지는 역할로 인식하게 되었습니다.

앞으로는 PostgreSQL 기반 운영 경험을 더욱 확장하여
문제가 발생하기 전에 예방할 수 있는 구조와 운영 체계를 설계하는
신뢰 중심 DBA로 성장하고자 합니다.
